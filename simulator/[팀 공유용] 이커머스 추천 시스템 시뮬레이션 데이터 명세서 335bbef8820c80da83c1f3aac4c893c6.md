# [팀 공유용] 이커머스 추천 시스템 시뮬레이션 데이터 명세서

본 데이터셋은 멀티모달 검색 및 Multi-Stage 추천 모델 학습을 위해 구축된 가상의 이커머스 행동 로그 데이터입니다. 단순 난수 생성이 아닌 **유저 페르소나와 시간 흐름(Markov Chain)이 반영된 현실적인 데이터**입니다.

---

### 1. 데이터셋 개요

- **상품 데이터:** 50,000건 (대/중/소 3단계 카테고리 적용)
- **유저 데이터:** 10,000명 (6가지 고유 페르소나 부여)
- **행동 로그:** 1,000,000건 (시간순 8:1:1 분할 적용)

---

### 2. 파일별 데이터 스키마 (Schema)

### 1. `products.csv` (아이템 메타데이터)

상품의 고유 정보입니다. 모델의 Item Feature로 활용됩니다.

| **컬럼명** | **데이터 타입** | **설명** | **예시** |
| --- | --- | --- | --- |
| `product_id` | string | 상품 고유 ID | P000001 |
| `category_L1` | string | 대분류 | 의류, 전자제품 |
| `category_L2` | string | 중분류 | 상의, 하의, 모바일, PC |
| `category_L3` | string | 소분류 | 티셔츠, 스마트폰, 노트북 등 |
| `price` | int | 상품 가격 (원) | 45000 |
| `price_tier` | string | 가격대 분류 (low/medium/high) | medium |

---

### 2. `users.csv` (유저 메타데이터)

고객의 고유 정보입니다. 모델의 User Feature로 활용됩니다.

| **컬럼명** | **데이터 타입** | **설명** | **예시** |
| --- | --- | --- | --- |
| `user_id` | string | 유저 고유 ID | U01234 |
| `persona` | string | 6가지 고객 성향 (구매 패턴 결정) | 가성비추구, 트렌드세터 등 |

---

### 3. `train_logs.csv`, `valid_logs.csv`, `test_logs.csv` (행동 로그)

유저가 상품과 상호작용한 시계열 로그입니다. 100만 건의 데이터를 시간순으로 80만(Train) / 10만(Valid) / 10만(Test) 건으로 엄격하게 분할했습니다.

| **컬럼명** | **데이터 타입** | **설명** | **예시** |
| --- | --- | --- | --- |
| `user_id` | string | 행동을 수행한 유저 ID | U01234 |
| `product_id` | string | 상호작용한 상품 ID | P000001 |
| `event_type` | string | 이벤트 종류 (search/view/cart/purchase) | view |
| `timestamp` | datetime | 행동 발생 시각 (YYYY-MM-DD HH:MM:SS) | 2023-04-15 14:30:22 |

---

**3. 모델링 시 반드시 고려해야 할 '데이터 비즈니스 로직'**
이 데이터에는 추천 AI가 반드시 찾아내야 할 숨겨진 규칙들이 심어져 있습니다.
• **페르소나별 취향 편향:** 유저는 자신의 페르소나에 맞는 타겟 카테고리(`category_L1`)를 80% 이상의 확률로 집중 탐색합니다. (예: '트렌드세터'는 의류 위주로 조회함)
• **이커머스 퍼널(Funnel) 비율:** 전체 로그의 비중은 `search`(~44%) →`view`(~44%) →`cart`(~9%) →`purchase`(~3%) 순으로 급격히 줄어드는 전이 확률(Markov Chain)을 따릅니다.
• **차별화된 CVR(전환율):** 장바구니(`cart`)에서 구매(`purchase`)로 넘어갈 때, '충동구매' 페르소나는 다른 유저보다 구매 확률이 2배 높고, '신중탐색' 페르소나는 구매 확률이 절반으로 떨어집니다.

---

### 4. 담당자별 활용 가이드

**To. 멀티모달 검색 리드 (Role 2)**

- `products.csv`의 `category_L1, L2, L3` 텍스트를 하나의 문장(예: "전자제품 모바일 스마트폰")으로 결합하여 텍스트 검색을 위한 CLIP 임베딩을 진행해 주세요.
- 상품 이미지가 필요한 경우, `product_id`와 DeepFashion/H&M 오픈소스 이미지를 매핑하여 이미지 인덱싱을 병행하시면 됩니다.

**To. 추천 시스템 모델링 리드 (Role 3)**

- **Candidate Generation (Two-Tower):** `users.csv`의 페르소나와 `products.csv`의 카테고리/가격을 임베딩 피처로 사용하세요.
- **Negative Sampling:** `train_logs.csv`의 노출/조회 후 장바구니에 담지 않은 상품, 혹은 랜덤 샘플링된 상품을 Implicit Negative 데이터로 활용하여 모델 변별력을 높여주세요.
- **최종 평가:** 반드시 `test_logs.csv` 데이터만을 사용하여 Recall, HitRate 등의 지표를 추출해야 Data Leakage 문제가 발생하지 않습니다.

 **To. 서빙 및 MLOps 리드 (Role 4)**

- 추천 로직에 필요한 "최근 조회 상품" 피처를 만들기 위해, `test_logs.csv`의 가장 마지막 시간대 데이터들을 유저별로 묶어 Redis에 초기 세팅해 두는 스크립트를 준비해 주세요.
